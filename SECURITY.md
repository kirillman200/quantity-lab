# Security policy

Report suspected vulnerabilities privately through the [GitHub Security Advisory form](https://github.com/kirillman200/quantity-lab/security/advisories/new). Do not open a public issue for an unpatched vulnerability.

Include the affected URL or component, the steps needed to reproduce the issue, the observed impact, and any suggested remediation. Do not include credentials, private project details, personal information, or data that does not belong to you.

Project Quantity Lab is a static, browser-only application. It has no login, server database, or public API. Calculation inputs can be stored in the current browser and encoded into a share link only when the user chooses those actions.

Supported releases are the current production deployment and the latest commit on `main`. Do not degrade availability, access other users' data, use social engineering, or perform destructive testing. Please allow a reasonable remediation window before public disclosure.
